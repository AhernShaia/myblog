Docker 概述

Why?

  • 上線流程繁瑣
    開發 → 測試 → 申請資源 → 審批 → 部署 → 測試…等環節
  • 資源利用率低
    普遍服務器利用率低,造成過多浪費
  • 擴容/縮容不及時
    業務高峰期擴容流程繁瑣,上線不及時
  • 服務氣環境臃腫
    服務器臃腫,對後期維護、遷移造成諸多不便
  • 環境不一致

What?

  • 操作系統級的虛擬化技術
  • 依賴 linux 特性: Namespace(資源隔離), Cgroups(資源限制)
  • 簡單的應用程式打包工具

容器 VS 虛擬化

容器

  1. 容器提供一個獨立的環境,實踐環境隔離、資源限制
  2. 主要解決應用層問題,便於快速部署、高效管理

虛擬化

  1. 提供服務器資源利用率
  2. 完全隔離的環境
